Tree Permit Lead Discovery System

User Manual & Operations Guide

Automated pipeline for South Florida tree-related permit lead discovery,
enrichment, scoring, and daily digest reporting.

v2.0 — April 2026

Document generated:

1. System Overview

The Tree Permit Lead Discovery System is a fully automated platform that monitors four South Florida government permit databases for tree removal, vegetation, landscape, and arborist permit filings. It continuously collects new permit data, enriches leads with property owner contact information, scores each lead by relevance and recency, and presents everything through a live interactive dashboard.

What the System Does

  1. Collects — Fetches permit records from 4 ArcGIS government APIs every 30 minutes during business hours
  2. Deduplicates — Prevents duplicate entries using permit number and normalized address matching
  3. Enriches — Adds property owner name, mailing address, phone, email, property value, and year built
  4. Scores — Ranks leads 0–10 based on permit type, recency, completeness, and configurable rules
  5. Geocodes — Adds map coordinates for leads missing location data
  6. Notifies — Sends daily email digests summarizing new lead activity

The pipeline monitors four government ArcGIS APIs. Each source publishes permit data on a weekday schedule — new records typically appear Monday through Friday during business hours.

Weekend & Holiday Behavior Government agencies do not file or publish new permits on weekends or federal holidays. Seeing zero new leads on Saturday, Sunday, or Monday mornings before noon ET is completely normal and not a system error.

9.1 Miami-Dade DERM Tree Permits

Source: Miami-Dade Department of Environmental Resources Management (DERM)

What it contains: All tree removal and alteration permits for unincorporated Miami-Dade County. Every record in this dataset is a tree permit — no keyword filtering required.

Special Characteristics

  • No native date field: DERM permits don't include a filing date. The system estimates dates by interpolating from permit sequence numbers at a rate of ~4.61 permits per business day.
  • Addresses may be delayed: Some DERM permits initially lack a facility address. The pipeline periodically re-checks the API and updates addresses when they become available.
  • Permit numbering: Format is TREE-02YNNNNN where Y = year digit, NNNNN = sequence. Example: TREE-02600309 = the 309th tree permit of 2026.

Annual volume: ~1,100–1,400 permits per year

9.2 Fort Lauderdale Building Permits

Source: City of Fort Lauderdale Building Permit Tracker

What it contains: General building permits filtered for tree and landscape-related categories:

  • Landscape Tree Removal-Relocation Permit (~595/year)
  • Landscape Installation Permit (~422/year)
  • ROW Landscaping Permit (~303/year)

Annual volume: ~1,321 relevant permits per year

Data includes: Contractor name and phone (often pre-populated), owner name, estimated cost

9.3 City of Miami Tree Permits

Source: City of Miami Tree Permits dataset

What it contains: Dedicated tree permit records for the City of Miami jurisdiction. All records are tree-related — no keyword filtering required.

Total records: ~6,000+ permits

Date field: Review Status Changed Date (indicates last status update)

9.4 City of Miami Building Permits

Source: City of Miami Building Permits Since 2014

What it contains: General building permits (217,000+), filtered for tree/vegetation keywords in the Work Items and Scope of Work fields.

Relevant volume: ~500 tree-related permits

Date field: Issued Date

10. Property Data Enrichment

For every lead in Miami-Dade County, the pipeline queries the Miami-Dade Property Appraiser ArcGIS API to add:

FieldDescription
Owner NameLegal property owner (TRUE_OWNER1, TRUE_OWNER2)
Owner Mailing AddressWhere the owner receives mail (may differ from property address)
Property TypeLand use description (Residential, Commercial, Vacant, etc.)
Property ValueCurrent total assessed value
Year BuiltConstruction year of primary structure
Folio NumberOfficial county property identifier

Lookup Strategy: The system first tries to match by folio number (highest accuracy), then falls back to address matching.

Free Public Data Property Appraiser data comes from a free public ArcGIS API — no API keys or per-call costs. The system can look up any Miami-Dade County property at no cost.

11. Contact Enrichment

The system uses a multi-step enrichment cascade to find owner and contractor contact details (phone numbers and email addresses):

Step 1: Check local JSON cache (FREE — git-backed files) └─ Found? → Use cached data, skip API └─ Not found? → Continue... Step 2: Enformion Person Search (~$0.25/call) └─ Search by owner name + state + address └─ Returns: phone, email, address └─ Found phone? → Done Step 3: Enformion Business Search (~$0.25/call) └─ For LLC/Corp owners, search by company name + state └─ Returns: business phone, officers └─ Found phone? → Done Step 4: Florida Sunbiz Officer Lookup (FREE) └─ For FL businesses, look up registered officers └─ Re-search Enformion by officer name └─ Found phone? → Done Step 5: Trust Name Parsing (FREE) └─ Extract person name from trust entities └─ Re-search Enformion by extracted name └─ Found phone? → Done Step 6: Property Reverse Address Search (~$0.25/call) └─ Search by property address only (no name needed) └─ Returns: current resident phone/email └─ Last resort when all name-based searches fail

Cost Controls

  • Cache-first: Every API response is saved as a JSON file. Subsequent lookups read from cache at zero cost.
  • Call limit: Maximum 5 new API calls per pipeline run (configurable).
  • Production only: Paid API calls are restricted to the production repository. Development environments can only read cached data.
  • Recent leads only: Contact enrichment applies to leads from the last 30 days only.

12. Email Digests

The system sends automated email reports summarizing pipeline activity:

Daily Digest

  • Schedule: Sent daily at 11:30 AM ET
  • Condition: Only sent when there are genuinely new leads inserted since the last digest
  • Contents: Summary of new leads by source, pipeline health status, link to dashboard
  • Recipients: All addresses in the notification list + active email subscribers from the database
No Spam Guarantee The email digest is never sent when there are zero new leads. If the pipeline had no new insertions (common on weekends and slow government days), no email is generated.

Welcome Report

When a new subscriber signs up through the dashboard's Email Reports tab, they receive an immediate 7-day deep-dive report with all leads from the past week.

14. Configuring Scoring Rules

The Scoring Rules tab in the dashboard allows administrators to customize how leads are scored.

How It Works

  1. Open the Scoring Rules tab
  2. Adjust bonus and penalty values for each factor
  3. Click Save
  4. Changes take effect within 5 minutes (cache refresh)
  5. All existing leads are re-scored on the next pipeline run

Available Parameters

  • Permit type bonuses (tree removal, vegetation, landscape, etc.)
  • Recency tier thresholds and bonuses (how many days = "hot" vs. "warm" vs. "aging")
  • Status penalties (e.g., intended decision, corrections required)
  • Property characteristics bonuses (large parcel, contact info present)
  • Staleness penalty threshold (days until a lead loses points)
Defaults Are Reasonable If the scoring rules table is empty or the database is unreachable, the system falls back to built-in defaults that work well for general tree service lead discovery.

18. FAQ & Troubleshooting

Why are there no new leads today?

Most likely because it's a weekend, Monday morning, or a government holiday. All four sources publish permits only during government business hours. The pipeline is working — there simply aren't new filings to discover.

Why does a lead have no phone or email?

Contact enrichment depends on the property owner being findable in public records databases. Some owners are difficult to locate because they:

  • Own through an LLC or shell corporation with no published contact info
  • Use a trust or holding company as the legal owner
  • Have an out-of-state or international address
  • Are a government entity or institutional owner

The system tries six different lookup methods (see Contact Enrichment) before giving up.

Why does a DERM lead show no address?

DERM sometimes publishes permits without a facility address initially. The system automatically re-checks the API every day at 8 AM ET and updates addresses when they become available. Leads with no address receive a −3 scoring penalty.

Why do lead scores change over time?

Scores include recency components. As a lead ages, its recency bonus decreases (e.g., a "hot" lead at +3 becomes "warm" at +1 after 30 days). Scores are recalculated at the end of every pipeline run.

The dashboard login page loads but I can't log in.

Verify that the Cloudflare Worker is deployed and the secrets (DASHBOARD_USERNAME, DASHBOARD_PASSWORD) are correctly set. Check the Worker's logs in the Cloudflare dashboard for errors.

I see "Pipeline lock held by another process" in the logs.

This means a previous pipeline run crashed without releasing its advisory lock. Go to GitHub Actions, find the Force Clear Locks workflow, and run it (type "YES" to confirm).

How do I add a new email subscriber?

They can self-subscribe from the Email Reports tab in the dashboard. Alternatively, insert directly into the email_subscribers table in Supabase.

How do I change the pipeline schedule?

Edit the cron expression in .github/workflows/daily_pipeline.yml (the schedule key). Push the change to the production repository. The new schedule takes effect within a few minutes.
