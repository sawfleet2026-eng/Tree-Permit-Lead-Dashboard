⚠️
Key Concerns:
  1. No addresses on any recent record. All address fields have been NULL since ~2013. Geocoding via external parcel lookups or FOLIO-based enrichment is required.
  2. No date field. Dates must be estimated from WORK_GROUP_NUMBER using a calibrated business-day interpolation model. Accuracy is ±1–2 days within the current year.
  3. Still the single best source for countywide tree permits — every record is a tree permit, no filtering needed, and data is clearly current (309 new permits in 2026 alone, ~1,384 in 2025).

Overall Recommendation

All three sources are live, publicly accessible, contain tree permit records, and have data current within the last few days. Development can proceed with confidence. The City of Miami additionally offers a dedicated tree permits layer as a bonus data source.

⚠️ Biggest Concern / Risk Found

The DERM dataset (Source 1) has no address fields populated on any recent record — all address columns have been NULL since approximately 2013. It also has no date field — dates must be estimated from WORK_GROUP_NUMBER using interpolation. These are real engineering challenges that have been solved in our pipeline but should be noted as inherent limitations of the source data.

🔮 Unexpected Discoveries

  1. Fort Lauderdale returns massive duplicate records — 2,493 raw results for tree permits actually represent only ~73 unique PERMITID values. The ArcGIS MapServer dynamically generates duplicate rows for the same permit.
  2. City of Miami has a dedicated Tree Permits layer (Tree_Permits/FeatureServer/0) with 6,013 records that have clean addresses, lat/long coordinates, and review status — an excellent supplemental source not mentioned in the original project brief.
  3. DERM data is a "table" type (no geometry) rather than a spatial feature layer — it has no map coordinates, only tabular permit data. Geocoding must be performed externally.
